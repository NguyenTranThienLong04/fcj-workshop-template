### Event Highlights

#### Hacking Your Brain to Learn Like a Game Addict (Huynh Hoang Long)

Huynh Hoang Long began his presentation with a familiar question: why is it so easy to spend hours scrolling through TikTok, yet so difficult to stay focused on studying for even ten minutes?

- **Root cause analysis:** The brain naturally favors activities that require little effort while providing immediate rewards and constant curiosity. This is the same principle that social media platforms and online games successfully exploit.
- **The Dopamine Mechanism:** Dopamine is released not when a reward is received, but when the brain anticipates that a reward is about to arrive. This explains why TikTok's recommendation system and slot machine mechanics are so addictive.
- **"Mystery Reward Box" technique:** Prepare several small rewards, write each one on a separate piece of paper, and place them in a box. After every ten minutes of studying, randomly draw one reward. The anticipation of the next surprise helps maintain motivation through continuous dopamine stimulation.
- **Leveraging Loss Aversion:** Create a daily study streak similar to Duolingo's streak feature or daily login rewards in games. As the streak grows longer, people become more motivated to avoid breaking it.
- **Task Decomposition:** Divide large learning goals into small, manageable tasks—for example, studying one AWS service each day instead of trying to learn everything at once. This approach is similar to beginners breaking a 5 km run into shorter segments.
- **The 2-Minute Rule:** If a task can be completed within two minutes, finish it immediately instead of postponing it.
- **XP System:** Create a personal experience point (XP) system where each completed topic earns points that can later be exchanged for small rewards or personal treats.
- **Share knowledge instead of scrolling:** Rather than spending time endlessly scrolling through social media, turn newly learned knowledge into posts or articles. This not only reinforces learning but also helps build the experience needed to become a speaker at future FCAJ Community Day events.

#### Automated Prompt Engineering & Proptimizer on AWS (Nguyen Tuan Thinh)

Nguyen Tuan Thinh delivered his presentation entirely in English, focusing on effective communication techniques for working with modern AI models.

- **The problem with generic prompts:** Ambiguous prompts often lead to inaccurate responses, consume more tokens, increase API expenses, and make AI models more likely to generate hallucinated information.
- **Seven essential components of an effective prompt:**
  - **Role:** Specify the role the AI should assume (e.g., *"You are a senior career coach"*).
  - **Instruction:** Clearly describe the task the AI is expected to complete.
  - **Context:** Provide the necessary background information for better understanding.
  - **Input:** Supply the text or data that needs to be processed.
  - **Output Format:** Define how the response should be structured, such as JSON, Markdown, or bullet points.
  - **Examples:** Include sample inputs and outputs using Few-shot Prompting to guide the model's behavior.
  - **Constraints:** Set limitations such as language, response length, or other specific requirements.
- **Advanced prompting techniques:**
  - **Chain of Thought (CoT):** Encourage the AI to solve complex problems through step-by-step reasoning.
  - **Self-Consistency:** Generate multiple reasoning paths and choose the answer supported by the majority.
  - **Tree of Thought (ToT):** Allow the model to explore multiple reasoning branches using search strategies such as BFS and DFS to identify the best solution.
- **Token Economics:** Tokens are the processing units of LLMs. Vietnamese generally requires more tokens than English, and API pricing depends on both input and output token usage.
- **Proptimizer Solution Architecture on AWS:**
  - **Frontend:** A Next.js application hosted on Amazon S3 and globally distributed through Amazon CloudFront with Origin Access Control (OAC).
  - **Authentication:** Amazon Cognito handles user registration, login, and JWT token generation without requiring a custom authentication system.
  - **API & Backend:** Amazon API Gateway receives incoming requests and forwards them to AWS Lambda for serverless business logic execution.
  - **AI Core:** Amazon Bedrock provides secure access to Foundation Models such as Claude and GPT without managing infrastructure.
  - **Storage:** Amazon DynamoDB stores prompt history and user configurations, while Amazon S3 stores static assets.
  - **Monitoring:** Amazon CloudWatch is used to monitor Lambda logs and API Gateway performance metrics.
  - **Product Feature:** Proptimizer is a browser extension that allows users to highlight any text on a webpage, automatically optimize it into a high-quality prompt, and use it directly with ChatGPT or Gemini.
#### Foundation Knowledge & the Role of AI for Freshers (Nguyen Duy Khang)

Nguyen Duy Khang, a Solutions Architect at Cloud Kinetics with around three years of industry experience, shared practical insights from a recruiter's perspective.

- **Prioritize strong fundamentals:** Technology evolves rapidly, but foundational knowledge remains valuable. Employers do not expect candidates to master every AWS service; instead, they value a deep understanding of the core services and a well-developed **thought process**.
- **AI as an amplifier:** AI does not replace software developers—it enhances their existing capabilities. Skilled engineers become more productive, while those with weak fundamentals expose their limitations more quickly.
- **AI can support your work, but not replace understanding:** Candidates who simply rely on AI-generated answers struggle when requirements change, whereas those who understand the concepts can adapt with confidence.
- **There is no universally correct architecture:** The key factor is the reasoning behind your design decisions. Clear analysis and well-supported arguments are often more important than memorizing standard solutions.

#### BMAD Method — Structured AI-Driven Software Development (Thao Nguyen)

BMAD (Breakthrough Method for Agile AI-Driven Development) is an open-source framework that introduces structure and discipline into AI-assisted software development, replacing the unorganized approach commonly known as "vibe coding."

- **Agentic Roles:** AI agents are assigned specialized responsibilities such as Product Manager, Architect, Developer, QA, and Scrum Master, with each role covering a specific stage of the SDLC.
- **Two Core Phases:**
  - **Phase 1 - Agentic Planning:** Prepare essential artifacts including PRDs, system architecture designs, and user stories before implementation begins.
  - **Phase 2 - Context-Engineered Development:** Use the documentation created during Phase 1 as context to help AI generate consistent, maintainable, and accurate code.
- **Practical Benefits:**
  - Minimize AI hallucinations through cross-validation among specialized agents.
  - Produce well-documented codebases that are easier to maintain, audit, and extend.
  - Suitable for projects ranging from personal applications to enterprise-scale systems.
  - Can be installed via CLI (`npx bmad-method install`) and integrated with tools such as Cursor, Claude Code, and other AI-powered development environments.
